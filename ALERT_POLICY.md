# Політика алертів DPS-Monitor

> Документ визначає правила — які вхідні документи та події генерують сповіщення.
> Оновлювати при додаванні нових типів клієнтів.

---

## 1. Типи клієнтів

| Тип | Система оподаткування | Статус |
|-----|-----------------------|--------|
| ФОП — єдиний податок | Спрощена (1–3 група) | ✅ Діє |
| ФОП — загальна система | Загальна (ПДФО + ВЗ) | 🔲 Не реалізовано |
| ЮО — єдиний податок | Спрощена (3 група) | 🔲 Не реалізовано |
| ЮО — загальна система | Загальна (прибуток + ПДВ) | 🔲 Не реалізовано |

---

## 2. Алерти по змінах бюджету (порівняння кешу)

Виявляються в `detectAlerts()` (`lib/dps/alerts.ts`) — щоденно через cron.

| Подія | Тип алерту | Всі типи клієнтів |
|-------|------------|-------------------|
| Борг з'явився (0 → > 1 грн) | `new_debt` | ✅ |
| Борг збільшився (> 1 грн) | `debt_increased` | ✅ |
| Борг погашено | `debt_cleared` | ✅ |
| Переплата змінилась (> 1 грн) | `overpayment_changed` | ✅ |
| Статус платника змінився | `status_changed` | ✅ |

---

## 3. Алерти по вхідних документах (BOTB0501 — нарахування зобов'язань)

Документ `cdoc = BOTB0501` — автоматичне повідомлення системи ДПС про нараховані зобов'язання.
Текст містить рядки вигляду: `по <КОД> <РАХУНОК> <НАЗВА ПОДАТКУ> - <СУМА>`.

### 3.1 ФОП на єдиному податку

**Ігнорувати** (не генерувати алерт) якщо документ містить ТІЛЬКИ ці податки:

| Код бюджетної класифікації | Назва | Причина ігнорування |
|---------------------------|-------|---------------------|
| `71040000` / префікс `70` | ЄСВ (Єдиний соціальний внесок) | Рутинне нарахування |
| `18050400` / префікс `50` | Єдиний податок з фізичних осіб | Рутинне нарахування |
| `11011700` / префікс `30` | Військовий збір (ФОП на спрощеній) | Рутинне нарахування |

**Генерувати алерт** `new_document` якщо документ містить будь-який ІНШИЙ податок
(наприклад: ПДФО, ПДВ, земельний, рентна плата, акциз тощо).

### 3.3 ЮО на єдиному податку — TBD

## 4. Алерти по вхідних документах (інші типи)

| `cdoc` | Назва | Алерт | Пріоритет | Умова |
|--------|-------|-------|-----------|-------|
| `J1499202` | Квитанція №2 | ❌ | — | Технічна квитанція |
| `BOTB0501` | Нарахування зобов'язань | ⚠️ | Залежить від правил п.3 | |
| `F1419104` | Довідка про доходи (ДРФО) | ✅ | 🟡 Середній | Завжди |
| `D0300201` з ІПК в індексі | Індивідуальна податкова консультація | ✅ | 🟡 Середній | `name` або `text` містить `/ІПК/` |
| `D0300201` — "не за адресою" | Помилковий документ | ❌ | — | `text` містить "не за адресою" |
| `D0300201` — Світовий банк (`csti: 9999`) | Спам/опитування | ❌ | — | `csti === 9999` |
| `PDI*` | Запит від ДПІ | ✅ | 🔴 Високий | Завжди |

## 5. Коди відправників (csti)

| Код | Опис |
|-----|------|
| `2030` | Центральний вузол ДПС (технічні квитанції) |
| `2040` | Місцева ДПІ (листи, запити) |
| `9900` | Автоматична система ДПС (BOTB, ІПК) |
| `9999` | Системні/рекламні повідомлення — ігнорувати |

---

## 6. Кеш документів

Для виявлення нових документів cron зберігає в `dps_cache` запис з `data_type = 'documents'`:
```json
{ "ids": ["id1", "id2", ...], "count": N, "fetched_at": "..." }
```
Нові документи = id присутні в свіжому запиті, але відсутні в кеші.
